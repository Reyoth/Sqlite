SqLite

SQLite est une database compacte. Avec toutes les fonctionnalités activées, Sa taille est inférieure à 500KiB, ce qui en fait une base de données de choix pour les appareils à mémoire limitée comme les téléphones cellulaires, les PDA, lecteurs MP3,...

Elle est simple à implémenter et semble être le meilleur choix pour qui veut une application avec base de données embarquée.

Ses points forts

Largement déployée

C’est la base de données la plus largement déployée au monde, on la retrouve entre autre dans des projets de grande envergure comme Mozilla Firefox, Microsoft Windows 10, Apple Mac OS-X.

Légère

En fonction des fonctionnalités activées, la taille de la librairie varie de 300KiB à 500KiB.

Performante

Grace à sa conception en arbre B et au fait que SqLite soit une database embarquée, elle est extrèmement performante.

OpenSource

Le code source de SQLite est dans le domaine public ce qui veut dire que n'importe qui peut copier, modifier et redistribuer son code.

Grande communauté

Son développement est pris en charge par une équipe internationale de développeurs qui travaillent dessus à plein temps afin d'améliorer sa fiabilité et ses performances.

Facile à implémenter

SqLite est supportée par la plupart des languages de programmation et du fait que la base de donnée est embarquée, elle est simple à integrer dans n'importe quel projet.

En savoir plus

1. Origine

Le projet SQLite a été conçu le 09/05/2000 par D. Richard Hipp qui travaillait pour General Dynamics sur contrat avec la US Navy. Hipp concevait des logiciels utilisés à bord des missiles guidés.
Les objectifs de conception de SQLite étaient de permettre à un programme de fonctionner sans installer de système de gestion de base de données ou nécessitant un administrateur de base de données.

2. Le modèle embarqué

Ce modèle a pour avantage de faire une application indépendante qui intègrera directement le moteur de base de données. Cela permet d’avoir de petites bases de données très performantes car stockées en local, d’éviter des échanges clients serveurs qui peuvent être mal sécurisé tout en restant facile à implémenter.

3. Architecture

Son architecture est assez simple. L’application développée devra passer par une API pour accéder au compilateur SQLite.
Le compilateur est composé d’un séquenceur qui se chargera de diviser les commandes données en parties traitables séparément, d’un analyseur syntaxique qui s'occupe de décomposer les requêtes en différents objets et enfin d’un générateur de code qui crée un bytecode.
Le bytecode obtenu est envoyé à La machine virtuelle qui interprète ces instructions. Elle fait ensuite appel au gestionnaire d'arbre B au niveau des ressources qui repose sur des couches de plus bas niveau dont le cache et sa couche d'abstraction.
